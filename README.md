# Kenya Fuel Intelligence

A modern, responsive commodity intelligence MVP for tracking fuel prices in Kenya. Built with **Next.js 15**, **React 19**, **TypeScript**, **Tailwind CSS**, **SQLite (better-sqlite3)**, and **Recharts**, styled with a dark "financial terminal" aesthetic similar to Bloomberg/TradingView.

## Features

- **Home Dashboard** — current Super Petrol, Diesel, and Kerosene prices (KES/Litre), previous month price, percentage change, and last-updated date.
- **Historical Price Trends** — interactive 24-month line chart with toggleable series, plus month-over-month percentage-change bar charts and a full data table.
- **Market Insights** — AI-style commentary cards explaining recent price movements, stored in and served from the database.
- **News & Events** — a timeline of EPRA announcements, global oil market events, currency movements, and OPEC news with impact levels.
- **Dashboard Statistics** — current prices for all three fuels plus the 24-month high and low.
- **REST API** — `/api/prices`, `/api/history`, `/api/news`, `/api/insights`.

## Database Schema

| Table              | Purpose                                                            |
|--------------------|---------------------------------------------------------------------|
| `commodities`      | Super Petrol, Diesel, Kerosene — name, slug, unit, color           |
| `locations`        | Nairobi, Mombasa, Kisumu                                            |
| `commodity_prices` | Monthly price per commodity/location (24 months of mock history)  |
| `news_events`      | Fuel-related news with title, description, date, impact level      |
| `insights`         | AI-style market commentary linked to a commodity and period        |

The schema is created automatically (`src/lib/db.ts`) and seeded with sample data via `scripts/seed.js`.

## Getting Started

### Prerequisites

- Node.js 18.18+ (Node 20 LTS recommended)
- npm

### Installation & Setup

```bash
# 1. Install dependencies
npm install

# 2. Seed the SQLite database with sample data
npm run db:seed

# 3. Run the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

> **Note:** `npm run db:seed` creates `data/fuel.db` with 24 months of mock price history, 3 locations, 8 news events, and 5 market insights. Re-running it resets the database to a fresh state. The `npm run build` script also runs the seed automatically so the database exists before production builds.

### Available Scripts

| Command           | Description                                  |
|--------------------|-----------------------------------------------|
| `npm run dev`      | Start the development server                  |
| `npm run build`    | Seed the database and build for production    |
| `npm run start`    | Start the production server (after build)     |
| `npm run db:seed`  | (Re)seed the SQLite database                  |
| `npm run lint`     | Run ESLint                                     |

## Deployment (Vercel)

This project is ready for direct deployment to Vercel:

1. Push the repository to GitHub/GitLab/Bitbucket.
2. Import the project in Vercel.
3. Vercel will run `npm install` and `npm run build` automatically. The `build` script seeds the SQLite database before building so the app has sample data.

### Notes on SQLite on Vercel

Vercel's serverless filesystem is **read-only at runtime except for `/tmp`**, but the bundled `data/fuel.db` file (created during the build step) is included in the deployment and can be read at runtime — this MVP only performs reads (and the one-time seed during build), so it works out of the box on Vercel's serverless functions. If you need to write data at runtime in production, swap `better-sqlite3` for a hosted database (e.g. Turso, Postgres on Neon/Supabase) — the `src/lib/queries.ts` data-access layer is structured so this is a drop-in change.

## Customization

- **Mock data** — edit `scripts/seed.js` to change base prices, volatility, locations, news events, or insights, then run `npm run db:seed`.
- **Styling** — color palette and design tokens live in `tailwind.config.ts` and `src/app/globals.css`.
- **Adding commodities** — insert new rows into the `commodities` table in `scripts/seed.js`; the dashboard, charts, and API automatically adapt to additional commodities returned by the queries (UI components may need minor extension for more than 3 series).
